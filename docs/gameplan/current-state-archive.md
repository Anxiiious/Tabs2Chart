- 2026-07-21 — **Resync-drift question actually settled, not just argued:** built a ~250-chord adversarial synthetic progression (ascending run, rest ≥1 bar, descending run with bigger steps, 30-chord plateau, section-marker reset, another ascending run with periodic octave leaps; 215/300 chords triggered a scoring override in the exploratory run). Compared each chord's actual emitted anchor lane against a baseline (the same sequence run as single notes, `k=1`, zero chord-scoring override involved). **Result: no drift, and it's structurally impossible, not just empirically absent** — (1) both actual and baseline lanes are always already-wrapped into 0-4 by `% 5`, so their difference is trivially bounded to `[-4,4]`, no unbounded quantity exists for error to accumulate in; (2) `_interval_to_step`'s step magnitude depends only on the real pitch interval, never on the cursor's absolute value, so a resync is a one-time phase-shift with no mechanism to bias later step size/direction. Empirically confirmed: front-half vs back-half mean deviation from baseline were statistically indistinguishable, no growth. Turned into a permanent regression test, `test_cursor_resync_does_not_drift_over_long_progression`. **63 tests pass, 0 xfail.**
